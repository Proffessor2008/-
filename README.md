Второй код(main) — объектно-ориентированная переработка, первый — оптимизированный и более «плоский» вариант.
🧠 Архитектурные отличия
Аспект	Первый код (оптимизированный)	Второй код (ООП)
Частицы	Обычные объекты (plain object), поля мутируются напрямую	Экземпляры класса Particle c методами .update(), .draw()
Векторная математика	Функции, работающие напрямую с координатами (x, y), без создания лишних объектов	Класс V (Vector2), каждый вызов метода создаёт новый объект или временные экземпляры
Связи	Функция drawConnections вне классов	Отдельный класс CM
Постобработка (grain)	Пререндеренный Canvas-шум, кешируется	Пересоздаёт ImageData и модифицирует пиксели каждый кадр
Resize	Мгновенная реакция	Debounce 150 мс (дёргается при растягивании окна)
